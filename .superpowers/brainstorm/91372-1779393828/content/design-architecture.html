Data flow per frame

Camera frame arrives (raw buffer)
  │
  ▼
For each of 4 digits:
  Derive 7 sample centers from 3 anchor points (a, g, b)
  │
  ▼
For each sample center:
  Average brightness in (2*radius+1)² pixel patch
  │
  ▼
Auto-threshold = (min_brightness + max_brightness) / 2
  │
  ▼
Classify 7 segments → ON/OFF bitmask
  │
  ▼
Lookup bitmask in truth table → digit 0–9 (or -1 = unknown)
  │
  ▼
value = d[0]*1000 + d[1]*100 + d[2]*10 + d[3]
  │
  ▼
Publish via ESPHome sensor (unit: mm)

Camera format requirement

Component needs raw pixel access. User must configure esp32_camera with grayscale pixel format for direct byte access. RGB565 also supported (convert to grayscale on read). JPEG format not supported — too expensive to decode on ESP32.

Geometry derivation (3 anchors → 7 sample points)

Given: a=(ax,ay), g=(gx,gy), b=(bx,_)
  x_left  = 2*ax - bx       // mirror b around a
  x_right = bx
  y_top   = ay
  y_mid   = gy
  y_bot   = 2*gy - ay       // reflect top through mid
  y_th    = (ay + gy) / 2   // top-half vertical midpoint
  y_bh    = (gy + y_bot)/2  // bottom-half vertical midpoint

Sample centers:
  a → (ax,   y_top)
  b → (bx,   y_th)
  c → (bx,   y_bh)
  d → (ax,   y_bot)
  e → (x_left, y_bh)
  f → (x_left, y_th)
  g → (gx,   y_mid)
